Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Essential Features

Request URL HTTP Method Request body data format Response body data format
Welcome page / GET None Welcoming text
Documentation /documentation.html GET None Documentation about the API
Return a list of all the movies /movies GET None JSON object holding data about all the movies
Return data about a single movie by title /movies/:Title GET None JSON object holding data about a single movie by title
e.g.
{ "_id" : ObjectId,
"Title" : "Title",
"Description" : "Description",

Return data about a genre by name /genres GET None JSON object holding data about all genres

e.g.

"Name" : "Name",

Return a list of all the directors /directors GET None JSON object holding data about all directors

e.g.
{ "_id" : "ObjectId",
"Name" : "Name",
"Bio" : "Bio",

Allow new users to register /users POST A JSON object holding data about the new user
e.g.
{ Username: String,
Password: String,

Birthday: Date } 		A JSON object holding data about the user who registered
e.g.

Allow users to update their user info /users/:Username PUT A JSON object holding data about the user´s update
e.g.
{ Username: String,
Password: String,
Email: String,
Birthday: Date } 		A response that contain the user´s updated information
e.g.
{ "Username": "Username",

"Email": "Email@Email.com",
"Birthday": "1993-01-01" }
Return a list of all the users /users GET None JSON object holding data about all the users
Return data about a single user by username /users/:Username GET None JSON object holding data about a single user by username
e.g.
{ "Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1993-01-01" }
Allow users to add a movie to their list of favorites /users/:Username/favorites/:_id POST A JSON Object holding data about the movies to add to the favorites list A response containing that the user added a movie to the favorites list
e.g.
{ "FavoritMovies": [movieID],
"Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1993-01-01" }
Allow users to remove a movie from their list of favorites /users/:Username/favorites/:_id DELETE None A text message indicating whether the movie was successfully removed from favorites.
